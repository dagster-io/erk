# Fix: Remove stale branch_slug forwarding in setup_impl that crashes `erk exec setup-impl --issue`

## Context

Commit `64bb34dd4` ("Move branch slug generation to skill layer") added `--branch-slug` to both `setup_impl` and `setup_impl_from_issue`. Later, commit `41ba25ed2` ("Remove plan ID encoding from branch names") removed the issue-backend code path from `setup_impl_from_issue`, which was the only path that used `branch_slug`. The `branch_slug` parameter was removed from `setup_impl_from_issue`'s Click signature, but `setup_impl.py` still passes it via `_handle_issue_setup` → `ctx.invoke(setup_impl_from_issue, branch_slug=branch_slug)`.

This causes a `TypeError` crash on **every** `erk exec setup-impl --issue <N>` invocation (including auto-detect paths), because Click raises `TypeError: setup_impl_from_issue() got an unexpected keyword argument 'branch_slug'`.

The O-prefix in branch names (`plnd/O{N}-{slug}-{timestamp}`) is correctly generated by `generate_planned_pr_branch_name()` in both `plan_save.py` and `one_shot_dispatch.py`. The setup-impl crash prevents the implementation flow from working at all, which makes it appear that the branch naming is broken.

## Fix

Remove the stale `branch_slug` parameter from `setup_impl.py` since `setup_impl_from_issue` no longer accepts or needs it (planned-PR setup checks out an existing branch rather than creating one).

### 1. `src/erk/cli/commands/exec/scripts/setup_impl.py`

- Remove `--branch-slug` Click option (lines 161-165)
- Remove `branch_slug` from `setup_impl()` parameter list
- Remove `branch_slug` from all `_handle_issue_setup()` call sites (lines 184, 212, 256)
- Remove `branch_slug` parameter from `_handle_issue_setup()` definition (line 272)
- Remove `branch_slug=branch_slug` from `ctx.invoke(setup_impl_from_issue, ...)` (line 295)

### 2. `tests/unit/cli/commands/exec/scripts/test_setup_impl.py`

- Remove any `--branch-slug` usage from test invocations
- Verify tests pass with the parameter removed

## Verification

1. Run setup_impl tests: `pytest tests/unit/cli/commands/exec/scripts/test_setup_impl.py -v`
2. Run setup_impl_from_issue tests: `pytest tests/unit/cli/commands/exec/scripts/test_setup_impl_from_issue.py -v`
3. Run naming tests (confirm O-prefix still works): `pytest tests/core/utils/test_naming.py -v`
4. Run fast-ci
