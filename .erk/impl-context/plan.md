# Fix flaky timestamp test in `test_output_modes.py`

## Context

`test_create_with_json_and_plan_file` fails when the test run crosses a minute boundary. The worktree name includes a minute-level timestamp (`%y-%m-%d-%H%M`), generated by `datetime.now()` inside `ensure_unique_worktree_name_with_date()`. The test then calls `get_current_date_suffix()` (also `datetime.now()`) to compute the expected name — if a minute ticks between the two calls, they disagree. This also violates the architecture tripwire against bare `datetime.now()` in production code.

A second test `test_create_with_stay_and_plan_file` has the same issue.

## Approach

Thread `ctx.time.now()` from the CLI command into the naming function, then use `FakeTime` in tests for deterministic timestamps.

## Changes

### 1. `packages/erk-shared/src/erk_shared/naming.py`

Add `now` parameter to `ensure_unique_worktree_name_with_date` and the deprecated `ensure_unique_worktree_name`:

```python
def ensure_unique_worktree_name_with_date(
    base_name: str, worktrees_dir: Path, git_ops, *, now: datetime | None = None
) -> str:
    date_suffix = (now or datetime.now()).strftime(WORKTREE_DATE_SUFFIX_FORMAT)
    ...

def ensure_unique_worktree_name(
    base_name: str, worktrees_dir: Path, git_ops, *, now: datetime | None = None
) -> str:
    return ensure_unique_worktree_name_with_date(base_name, worktrees_dir, git_ops, now=now)
```

### 2. `src/erk/cli/commands/wt/create_cmd.py`

Pass `now=ctx.time.now()` at both call sites:

- **Line 173**: `name = ensure_unique_worktree_name_with_date(name, repo.worktrees_dir, ctx.git, now=ctx.time.now())`
- **Line 734**: `name = ensure_unique_worktree_name(name, repo.worktrees_dir, ctx.git, now=ctx.time.now())`

### 3. `tests/commands/workspace/create/test_output_modes.py`

In both `test_create_with_json_and_plan_file` and `test_create_with_stay_and_plan_file`:

- Import `FakeTime` and pass it via `build_context(time=FakeTime())`
- Compute `expected_name` from `FakeTime.DEFAULT_FAKE_TIME` instead of calling `get_current_date_suffix()`

```python
from erk_shared.gateway.time.fake import FakeTime
from erk_shared.naming import WORKTREE_DATE_SUFFIX_FORMAT

fake_time = FakeTime()
test_ctx = env.build_context(git=git_ops, local_config=local_config, repo=repo, time=fake_time)
...
date_suffix = FakeTime.DEFAULT_FAKE_TIME.strftime(WORKTREE_DATE_SUFFIX_FORMAT)
```

### 4. `tests/commands/workspace/create/conftest.py`

Remove `get_current_date_suffix()` if no other tests use it. (Check first — if other tests import it, keep it but it's no longer needed by the fixed tests.)

## Files modified

- `packages/erk-shared/src/erk_shared/naming.py` — add `now` parameter
- `src/erk/cli/commands/wt/create_cmd.py` — pass `ctx.time.now()` at lines 173, 734
- `tests/commands/workspace/create/test_output_modes.py` — use `FakeTime` for deterministic timestamps
- `tests/commands/workspace/create/conftest.py` — remove `get_current_date_suffix` if unused
